# UI & I18n

## Layout

### HTML Root Configuration
Always set `dir` and `lang` on the `<html>` element based on the active locale:

```html
<html lang="ar" dir="rtl">
<html lang="en" dir="ltr">
```

### Framework-Specific Direction Providers
- **[Radix UI](https://www.radix-ui.com/)**: Use `<DirectionProvider dir="rtl">` wrapper.
- **[shadcn/ui](https://ui.shadcn.com/)**: Follow their [RTL configuration guide](https://ui.shadcn.com/docs/rtl/).

## Writing Systems & Direction

Writing mode categories:
- **LTR** â€” Latin, Indic scripts (English, French, German, Hindi)
- **RTL** â€” Arabic, Persian, Hebrew
- **Bidirectional** â€” mixed Latin + Arabic
- **Vertical** â€” Asian scripts (Japanese, Chinese, Mongolian)

## `dir` Attribute

Controls inline direction (horizontal axis):
- `ltr` â€” left to right
- `rtl` â€” right to left
- `auto` â€” browser detects from first strong character

Use `dir="auto"` on individual content blocks (e.g., each paragraph in a text editor) for mixed-direction user-generated content.

### Styling Based on `dir`
Two approaches:
1. `:dir()` pseudo-class â€” uses **semantic** directionality. Works with CSS frameworks and preprocessors.
2. `[dir=ltr]` / `[dir=rtl]` attribute selectors â€” only matches elements with explicit `dir` attribute in HTML.

RTL styling guide: [rtlstyling.com](https://rtlstyling.com/posts/rtl-styling)

## `unicode-bidi` CSS Property

Controls the Unicode Bidirectional Algorithm for mixed-direction text:

| Value | Behavior |
|---|---|
| `normal` | Default â€” standard implicit rules |
| `embed` | Creates an embedding layer for inline elements |
| `isolate` | Fences off content from affecting surrounding layout |
| `bidi-override` | Forces strict ordering by `direction` property |
| `isolate-override` | Isolated fence + forced directional ordering inside |
| `plaintext` | Ignores CSS `direction`, detects from actual characters |

> âš ď¸Ź **Prefer the HTML `dir` attribute** over CSS `direction`/`unicode-bidi` in HTML documents. Only use CSS bidi properties for mapping bidi info in markup to CSS styles, not to control direction in HTML.

## `writing-mode` CSS Property (Block Direction)

| Value | Description |
|---|---|
| `horizontal-tb` | Default. Top-to-bottom block, horizontal text |
| `vertical-rl` | Right-to-left block, vertical text |
| `vertical-lr` | Left-to-right block, vertical text |
| `sideways-rl` | Vertical writing, text rotated sideways (RL) |
| `sideways-lr` | Vertical writing, text rotated sideways (LR) |

Writing system defaults:
- **Latin** â†’ `horizontal-tb` + LTR
- **Arabic** â†’ `horizontal-tb` + RTL
- **Mongolian** â†’ `vertical-lr` + LTR
- **Han (Chinese/Japanese)** â†’ either `horizontal-tb` or `vertical-rl`

Article: [CSS Writing Modes explained simply](https://24ways.org/2016/css-writing-modes/)

## Typography

### `font-family`
Two approaches for multi-language font support:
1. **Include all language fonts** â€” browser auto-detects which to use. âš ď¸Ź Performance risk with many language-specific fonts.
2. **Per-locale font loading** â€” load only the font needed for the active locale/direction.

### Relevant CSS Font Properties
- `font-weight` â€” stroke thickness / blackness
- `font-width` / `font-stretch` â€” horizontal scaling
- `line-height` â€” horizontal line spacing

### `letter-spacing`
âš ď¸Ź **Never apply `letter-spacing` to cursive scripts** (Arabic, Persian, and similar). It breaks the connected letterforms and is visually incorrect. This is a known CSS limitation â€” the spec explicitly warns against it.

Only appropriate for Latin scripts.

## Logical CSS Properties (Required for Bidirectional Support)

Use logical properties instead of physical ones so layout automatically mirrors for RTL/vertical writing modes.

### Mapping (Horizontal LTR baseline):
| Physical | Logical equivalent |
|---|---|
| `margin-left` | `margin-inline-start` |
| `margin-right` | `margin-inline-end` |
| `margin-top` | `margin-block-start` |
| `margin-bottom` | `margin-block-end` |
| `padding-left` | `padding-inline-start` |
| `text-align: left` | `text-align: start` |
| `border-right` | `border-inline-end` |

In RTL, `inline-start` = right, `inline-end` = left.
In vertical writing modes, block/inline axes swap.

Reference: [Digging Into CSS Logical Properties â€” Ahmad Shadeed](https://ishadeed.com/article/css-logical-properties/)

## Form Fields

### Vertical Fields
Use `writing-mode` + logical properties for:
- Range sliders, meters, progress bars
- Select elements
- Buttons
- Text fields

See: [MDN Vertical Controls Guide](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Writing_modes/Vertical_controls)

### Date Picker / Calendar
Three factors to control:
1. **Locale/language** â€” numbering system, weekday names
2. **Timezone** â€” what date is shown
3. **Calendar system** â€” Gregorian, Islamic, Buddhist, Coptic, Hebrew, Chinese, Indian, Japanese, Persian

Always configure all three, not just locale.

### Text Editor (i18n-aware)
Best practices:
- Use `Intl.Segmenter` for accurate character/word/sentence counts (essential for Arabic, Hindi, Thai)
- Apply `dir="auto"` on each block/paragraph
- Use logical CSS properties throughout
- Make each paragraph its own block element for per-block direction detection.

## HTML Attributes to Translate

- `title` â€” browser tooltip on hover
- `alt` â€” image alternative text
- `placeholder` â€” form field hint text
- `aria-*` â€” accessibility labels
- `data-*` â€” custom data attributes used for display
